存储、中断、总线与 I/O 系统

作者 : Walker Denial    发布于 : 2021-03-01

1 存储系统

对存储系统的基本要求是 大容量、高速度 和 低价格。

存储器容量：SM = W * L * m。

W：存储体的字长（单位是位或字节）

L：存储体的字数

m：并行工作的存储体数

TA：访问时间（速度）

TM：存储周期

Bm：频宽（也称带宽）

TA 是存储器从接收访存读申请至信息被读到数据总线上的时间，是处理机启动访存后必须等待的时间，它是确定处理机与存储器时间关系的一个重要参数。

TM 是连续启动一个存储体所需要的间隔时间，它一般总比 TA 大。

存储器价格包含了存储体及为该存储器操作所必须的外围电路的价格，可用总价格 C 和每位价格 c 来表示。有 SM 位的存储器每位价格 c = C/SM。

主存和辅存之间并不能构成完整的整体的系统为存储器系统。

1.1 领会并行主存系统的组成形式，极限频宽和实际频宽的计算

能并行读出多个 CPU 字的单体多字和多体单字、多体多字的交叉访问主存系统被称为并行主存系统。

Bm 是存储器可提供的数据传送速率，用每秒传送的信息位数或字节数衡量，又有最大频宽（或极限频宽）和实际频宽之分，最大频宽 Bm 是存储器连续访问时的频宽。单体的 Bm = W/TM，m 个存储体并行的最大频宽 Bm = W x m/TM。由于存储器不一定能满负荷工作，因此，实际频宽往往低于最大频宽。

1.2 领会通过使用并行主存的组成技术，提高主存实际频宽的可能性、局限性和发展存储体系的必要性

并行主存的组成技术：

  1. 多体单字；
  2. 多体单字；
  3. 多体多字；

提高模 m 值，是能提高主存系统的最大频宽的，单主存实际频宽并不是随 m 增大而线性提高，也就是说其实际效率并不像所希望的那么高。

因为程序的转移概率不会很低，数据发布的离散性较大，所以单纯靠增大 m 来提高并行主存系统的频宽是有限的，而且性能价格比还会随 m 的增大而下降。如果采用并行主存西戎仍不能满足速度上的要求，就必须从系统结构上进行改进，采用存储体系。

2 中断系统

CPU 中止正在执行的程序，转去处理随机提出的请求，待处理完后，再回到原先被打断的程序继续恢复执行的过程称为 中断。响应和处理各种中断的软、硬件总体称为 中断系统。

引起中断的各种事件称为 中断源。中断源 向中断系统发出请求中断的申请，称为 中断请求。同时可能有多个 中断请求，中断系统需按事先确定的中断响应优先次序对优先级高的中断请求给予响应。中断响应 就是允许其中断 CPU 现行程序的运行，转去对该请求进行预处理，包括保存好断点及其现场，调出有关处理该中断的中断服务程序，准备运行。这部分工作在多数计算机上都用交换新旧程序状态字 PSW 来实现。为了某种需要，中断系统也可以对中断请求进行屏蔽，使之暂时得不到响应。

2.1 中断分类和分级的目的，一般分为哪几类、哪几级

对中、大型多用途计算机，中断源数多达数十至数百个。如果为每个中断源单独形成入口，不仅硬件实现困难，代价大，就是在中断处理上也没有这种必要。因为不少中断源的性质比较接近，可以将它们归成几类。对每一类给定一个中断服务程序入口，再由软件分支转入相应的中断处理部分，就可以大大简化中断处理程序入口地址形成硬件。

在计算机中，中断可分为 内部中断、外部中断、软件中断 三类。

由于中断源相互独立而随机地发出中断请求，因此常常会同时发生多个中断请求。同一类的各中断请求的响应和处理的优先次序，一般不是由中断系统的硬件管理，而是由其软件或通道来管理的。而不同类的中断就要根据中断的性质、紧迫性、重要性以及软件处理的方便性把它们分成不同的级别。中断系统按中断源的级别高低来响应。

2.2 设置中断级屏蔽位的作用及中断嵌套的原则

中断的响应次序是在同时发生多个不同中断请求时，中断响应硬件中排队器所决定的响应次序。然而，中断的处理要由中断处理程序来完成，而中断处理程序在执行前或执行中是可以被中断的。这样，中断处理完的次序就可以不同于中断的响应次序。

中断响应的次序用排队硬件实现，次序是由高到低固定的。为了能根据需要，由操作系统灵活改变实际的中断处理程序，很多计算机都设置了中断屏蔽位寄存器，以决定某级中断请求能否进入中断响应排队器。只要能进入的，总是让高级别的优先响应。程序状态字中包含有中断级屏蔽位字段。只要操作系统把每一类中断处理程序的现行程序状态字中的中断级屏蔽位设置成不同的状态，就可以实现所希望的中断处理次序。

为保证中断嵌套时从哪儿来回哪儿去，均设置有一个返回地址堆栈，中断时将断点地址用硬件的方法自动压入栈顶保存，等中断返回后，再用硬件的方法从当前栈顶弹回到程序计数器，利用堆栈的后进先出实现正确的返回。另外，用户程序（目态程序）是不能屏蔽任何中断的，即用户程序的现行 PSW 中的中断级屏蔽位对各级中断都应当是开放的。

2.3 中断系统硬件功能分配状况

中断系统的功能包括中断请求的保存和清除、优先级的确定、中断断点及现场保存、对中断请求的分析和处理以及中断返回等。

中断响应及其次序由程序查询软件的方法改为用中断响应排队器硬件实现，中断的分析也由程序查询改为硬件编码，直接或经中断向量表间接形成各中断处理程序的入口地址，进而发展成每级中断经中断响应硬件形成该级中断程序状态字地址的入口，再把中断源的状况以中断码的形式经旧程序状态字告知中断处理程序。

3 总线设计

总线是用于互连计算机、CPU、存储器、I/O 接口及外围设备、远程通信设备间信息传送通路的集合。总线与其相配合的附属控制电路统称为总线系统。按信息传送功能、性能的不同，有数据线，地址线，命令、时序和中断信号等控制/状态，电源线，地线及备用线等。

数据线根数决定同时传送的数据位数，即数据通路宽度；地址线根数决定直接寻址的范围；控制/状态线决定总线的功能和使用能力；备用线用于系统功能的扩充。

3.1 专用和非专用总线的定义、优缺点及适用场合

只连接一堆物理部件的总线称 专用总线。

优点：

  1. 多个部件可以同时收/发信息，不争用总线，系统流量高；
  2. 通信时不用指明源和目的，控制简单；
  3. 任何总线的失效只会使连于该总线的两个部件不能直接通信，但它们仍可通过其他部件间接通信，因而系统可靠；

缺点：

  1. 总线数多；
  2. 总线较长时，成本高；
  3. 不利于系统模块化；
  4. 时间利用率低；

非专用总线 可以被多种功能或多个部件分时共享，同一时间只有一对部件可使用总线进行通信。

优点：

  1. 总线数少，造价低；
  2. 总线接口标准化、模块性强；
  3. 可扩充能力强，部件的增加不会使电缆、接口和驱动电路激增；
  4. 易用多重总线来提高总线的带宽和可靠性，使故障弱化；

缺电：

  1. 系统流量小，经常会出现争用的情况，使未获得总线使用权的部件不得不等待而降低效率；
  2. 共享总线失效会导致系统瘫痪；

3.2 非专用总线中 3 种总线控制方式的总线分配过程、优缺点、所增加的控制线线数及适用场合

总线控制结构基本几种在一起，无论是在连接到总线的一个部件中，还是在单独的硬件中，都称为集中式控制。而总线控制逻辑分散连到总线的各个部件时，就称为分布式总线控制。集中式总线控制的优先次序的确定可以有 串行链接、定时查询、独立请求 3 种不同的方式，也可以是它们的结合。

串行链接的优点：

  1. 选择算法简单，用于解决总线控制分配的控制线的线数少，只需要 3 根，且不取决于部件的数量；
  2. 部件的增减容易，只需简单地把它连到总线上或从总线上去掉即可，可扩充性好；
  3. 由于逻辑简单，容易通过重复设置提高可靠性；

缺点：

  1. 对「总线可用」先及其有关电路的失效敏感；
  2. 由于优先级是线连固定，不能由程序改变，不灵活；
  3. 因「总线可用」信号必须顺序脉动地通过各个部件，所以限制了总线的分配速度；
  4. 总线长度的限制，故增减或移动部件也受到限制；

定时查询的优点：

  1. 因计数器初值、部件号均可由程序制定，优先次序可用程序控制，灵活性强；
  2. 不会因某个部件失效而影响其他部件对总线的使用，可靠性高；

缺点：

  1. 控制线的线数较多；
  2. 可用贡献总线的部件数受限于定时查询线的线数，扩展性稍差；
  3. 控制较为复杂；
  4. 总线分配的速度取决于计数信号的频率和部件数，不能很高；

独立请求的优点：

  1. 总线分配速度快，所以部件的总线请求同时送到总线控制器，不用查询；
  2. 控制器可以使用程序可控的预定方式、自适应方式、循环方式或它们的混合方式灵活确定下一个使用总线的部件；
  3. 能方便地隔离实现部件的请求；

缺点：

  1. 控制线数量过大；
  2. 总线控制器要复杂的多；

3.3 同步和异步通信控制方式的通信过程、优缺点及适用场合

3.3.1 同步通信

部件间的信息传送由定宽、定距的系统时标同步。

优点：

  1. 信息的传送率高；
  2. 受总线长度的影响小；

缺点：

  1. 时钟在总线上的时滞可能会造成同步误差；
  2. 时钟线上的干扰信号易引起误同步；

3.3.2 异步通信

由于 I/O 总线一般是为有不同速度的许多 I/O 设备所共享，因此宜于采用异步通信。异步通信又分单项源控制和请求/回答双向控制两种。通信过程只由源或目的部件之一控制的称为单向源控式或单向目控式，而由源和目的共同控制的称为请求/回答双向控制。

单向源控式优点：

  1. 简单；
  2. 高速；

缺点：

  1. 没有来自目的部件指明传送是否有效的回答；
  2. 不同速度的部件间通信比较困难，部件内需设置缓冲器以缓存来不及处理的数据；
  3. 效率低，高速部件难以发挥其效能；
  4. 要求「数据准备」干扰小，否则易误认成有效信号；
  5. 不能保证下一个数据传送之前让所有数据下和控制线的电平信号恢复成初始状态，从而可能造成错误；

3.4 数据宽度的定义、分类及优缺点。数据宽度与数据通路宽度的不同

数据宽度是 I/O 设备取得 I/O 总线后所传送数据的总量。

数据宽度有单字（单字节）、定长块、可变长块、单字加长块和单字加长可变长块等之分。

数据通路宽度是数据总线的物理宽度，即一个时钟周期所传送的信息量。

4 输入/输出系统的基本概念

I/O（输入/输出）系统包括输入/输出设备、设备控制及与输入/输出操作有关的软、硬件。

4.1 高性能多用户计算机系统中，I/O 系统应当面向操作系统设计

输入/输出系统结构设计的好坏会直接影响计算机系统的性能，不仅影响输入/输出速度，各用户从程序输入到运算结果输出的时间，CPU、主存的利用率，还会影响到整个 I/O 系统的兼容性、可扩展性、综合处理能力和性能价格比等。

4.2 I/O 系统的 3 种方式，I/O 处理机的两种形式

输入/输出系统的发展经历了 3 个阶段，相对应于 3 种方式，即程序控制 I/O（包括全软件的、程序查询的、中断驱动的）、直接存储器访问（DMA）及 I/O 处理机方式。它们可分别用于不同的计算机系统，也可用于同一系统。

I/O 处理机方式，有通道（Channel）方式和外围处理机（PPU）之分。

5 通道处理机

5.1 通道处理机的输入/输出过程

先选择指定的通道、子通道，如它被连通且空闲时，就从主存中取出通道地址字，按通道地址字给出的通道程序首地址，从主存通道缓冲区取出第一条通道指令。经校验无误后，再选择相应设备控制器和设备。如该设备是被连着的，就向设备发启动命令。如果设备启动成功，用全「0」字节回答通道时，结束通道开始选择设备期。

5.2 通道的 3 种类型，相应采用的数据宽度及适用场合

根据通道数据传送期中信息传送方式的不同，可分为字节多路、数组多路和选择 3 类通道。

6 通道流量计算

6.1 通道的极限流量计算，外设对通道要求的流量计算

通道流量是通道在数据传送期内，单位时间内传送的字节数。它能达到的最大流量称通道极限流量。通道的极限流量与其工作方式、数据传送期内选择一次设备的时间 TS 和传送一个字节的时间 TD 的长短有关。

字节多路通道每选择一台设备只传送一个字节，其通道极限流量

fmax · byte = 1/(TS + TD)

数组多路通道每选择一台设备可传送完 K 个字节。如果要传送 N 个字节，就得分「N/K」次传送才行，每次传送都要选一次设备，通道极限流量。

fmax · block = K/(TS + KTD)

选择通道每选择一台设备就把 N 个字节全部传送完，通道极限流量

fmax · select = N/(TS +NTD)
