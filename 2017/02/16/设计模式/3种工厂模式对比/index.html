3种工厂模式对比

作者 : Walker Denial    发布于 : 2017-02-16

模式名称 优点 缺点
简单工厂模式（Simple Factory Pattern） 1、一个工厂类就可以实现功能；
2、外部无需关注如何获取所需对象		 1、每当添加或者删除一个对象时，需要修改 Factory 类，不符合开闭原则；
2、Factory 类职责过重，需要负责的东西很多
工厂方法模式（Factory Method Pattern） 1、新增产品时，无需修改 Factory 代码，符合开闭原则 1、每添加一个新产品，需要生成一个产品 class 和一个产品对应的 Facroty，会增加项目的复杂度；
抽象工厂模式（Abstract Factory Pattern） 1、新增产品族时无需修改 Factory 类，符合开闭原则，满足后期扩展的需求 1、新增产品等级结构的时候，需要修改 Factory 类，不符合开闭原则；
2、修改 Factory 的时候，由于 Factory 是抽象类，影响较大，如果系统比较庞大，会出现很多新问题
